### Kubernetes 표준 생태계로 편해진 IT 인프라 구축
-----
<div align="center">
<img src="https://github.com/user-attachments/assets/41db69a0-fe15-4c1c-9185-76d978cbdb7b">

1. 개발 : App 개발에서 배포까지 써야 되는 기술
2. 오케스트레이션 / 매니징 : App을 Micro-Service로 만들 떄 사용하면 좋은 기술
3. 플랫폼 / 런타임 : App을 클라우드로 올릴 때 주로 사용되는 기술
4. 프로비저닝 / 분석 : 실제 프로젝트에서 사용해야 되는 기술
5. 실제 프로젝트를 할 때 구조적 문제 (모니터링 / 문제)와 Kubernetes
   - 개발과 모니터링 시스템이 서로 엮일 수 밖에 없는 구조 : Kubernetes를 사용하면 Kubernetes 생태계에 있는 모니터링과 로깅 툴을 사용하면 해결 가능
   - 개발에서는 한 번도 써보지 않은 (개발 시스템을 위한) 모니터링 시스템을 만드는 구조 : 초기부터 바로 사용할 수 있도록 해줌
   - 오픈 시 개발 프로젝트와 서로 다른 범위의 App들을 모니터링하게 되는 구조 : App이 항상 자동으로 같아지는 범위로 모니터링 가능
